# PortSwigger Labs - Platform

**Lab - 1: Basic SSRF Against Local Server**
**Difficulty Level:** Apprentice

---

## Where Vulnerability Found?

- **URL:** `/product?productId=1`
- **Parameter:** `stockApi` (in the request generated by the Check Stock button)

---

## Steps

1. Clicked the Check Stock button and intercepted the request in Burp
2. Found the `stockApi` parameter in the request
3. Changed the value of `stockApi` → `http://localhost/admin`
4. The `/admin` panel opened
5. Deleted user Carlos → `/admin/delete?username=carlos`

---

## Working Payload

```
http://localhost/admin
http://localhost/admin/delete?username=carlos
```

---

## Why It Worked?

- The server itself was fetching `/admin`
- The admin panel was accessible only from localhost
- So a normal user couldn't access it, but the server could

---

## What I Learned?

- The `stockApi` parameter directly fetched a URL
- Since it was a server-side request, the firewall was bypassed
- Restricted panels on localhost can become accessible via SSRF
